JSR-303 Mechanism

Default Annotation

Customer class is a JSR-303 example.
There are standard annotations and Hibernate annotations. Hibernate annotation give your more convenience annotations to check data.
DateTimeFormat is a spring data formatting annotation, I event know this annotation before.
However, I will need to write a post about formatting. so I will try to explain in next Spring MVC tutorial.
I don’t want to explain every basic annotation by myself.
So, I found this article to explain.
But, I still make a list of them.
For JPA

  • @Null check if null
  • @NotNull check if not null
  • @AssertTrue check if true
  • @AssertFalse check if false
  • @Min(value) check if greater and equal than one value
  • @Max(value) check if lesser and equal than one value
  • @DecimalMin(value) check if greater and equal than one value, big value(Decimal)
  • @DecimalMax(value) check if lesser and equal than one value, big value(Decimal)
  • @Size(max, min) check string length, same as @Length
  • @Digits (integer, fraction) check digit format
  • @Past check if past date
  • @Future check if future date
  • @Pattern(value) check by regular expression

For Hibernate

  • @Email check email
  • @Length check string length
  • @NotEmpty check string is not empty
  • @Range example: @Range(min=0, max=10), check value range
