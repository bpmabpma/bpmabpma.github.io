Java controller

There are few annotations I want to mention. Controller, RequestMapping, Autowired, GetMapping, PostMapping, PutMapping, DeleteMapping, PathVariable.
Controller annotation you can tread it is a java bean. It means there is a Spring MVC controller bean.
You can also see another same type bean, like @Service, @Component. @Service means service layer bean. @Component means other types of java bean.
@RequestMapping is the routing annotation, and @GetMapping, @PostMapping, @PutMapping, and @DeleteMapping are the new type of routing annotation since Spring MVC 4.3.
@PathVaiable is for the path variable. Otherwise, @RequestParam is for Get parameters or submit value.
Model in the argument is an object stored the binding parameters, it can be an object or a map.
Because Spring MVC don’t support the multiple forms, it always returns one object at a time.

Then, let’s see how to data binding on Thymeleaf.
We use post method be our example. You can see this page on http://localhost:8080/posts/new

The th:action binds the submit path.
The th:object binds the return data and sending data.
The th:method let Thymeleaf can supports other http methods, like put and delete.
The th:field binds the field value of the return object.
