What the hell is Hexo?

Simplestly, it’s a blog framework. The most import reason which I choose Hexo is it supports markdown, and Hexo is based on Node.js.
And, it is very easy to use.

You also can read the Hexo offical website to understand how to install Node, npm, nvm.

Deploy to github page

If you want to deploy your blog to Github Page, Hexo is a greate chose to do it.
Otherwise, Hexo is not only support Github Page, it also suppot others, like Heroku, etc.
First you need to create a github page repository. You can following the instructions on Github Page.

Configure the deploy properties on __config.yml file.
GITHUB_ACCOUNT is your Github account. And branch(not sure) and message have their default values.
So, you don’t have to setup.

Install hexo-deployer-git, this is important step.
You need to install it to make sure you can deploy to github.

Deploy by your configuration.

Then, you can go to your Github Page to see the result.

Image on Hexo

You can use Hexo to put your images, and link them by markdown syntax.
Put the image file in the /source/images folder. And link by image syntax ![ALT_MESSAGE](images/IMAGE_FILE_NAME)
