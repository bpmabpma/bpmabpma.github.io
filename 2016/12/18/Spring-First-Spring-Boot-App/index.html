[Spring Boot] First Spring Boot App

2016-12-18
#Spring Boot

Actually, in 2016, I spent the most time to implement the website by Spring Boot.
So, I think it’s time to write into my blog. Although, I think there are plenty of blogs to introduce Spring Boot. Even though, I wish I can write done the whole implementation for my website.

I still will make some changes for my project.

Precondition

I think you should know following tools or knowledge before you start the project.

  • Java 8
  • Maven 3
  • IDE (Intellij IDEA or STS)

Java 8 is the newest version of Java, I use Oracle JDK for development, it supports a lot of new features, like Lambda.
I still use Maven, because it is enough for our purpose. Gradle is also a great build management tool, but any new thing will spend the time to learn. So, I still use Maven. IntelliJ is a great IDE, but it not free. STS(Spring Tool Suite) is also used in my work time. So, just chose the one you like.

Actually, I use a lot of libraries and tools. I will introduce later.
Followings are the tools I will use. But, I may miss some.
And, you will learn them later.

  • Spring Boot
  • Spring MVC
  • Hibernate & JPA & Spring Data
  • Spring Security
  • Wro4j
  • jQuery
  • Bootstrap
  • BEncrip
  • Apache Common Lang

These are very useful tools for a full stack Java developer. But, if you want split frontend and backend, you can use npm to implement frontend, or chose one famous frontend framework, like React, Angular, or Vue. But, for now, I still use jQuery.

Goal

  • Build a Hello World example for Spring Boot with Spring MVC

First App

This our example comes from (Building an Application with Spring Boot)[https://spring.io/guides/gs/spring-boot/]
My workflow is opening a Maven project fist.
Then, we copy the Maven configuration into pom.xml.
Next, we write our code.
Finally, we run our code and test it.

pom.xml

Configure our pom.xml.

I choice Thymeleaf to be our template engine. It has a very great skeleton.
Otherwise, the Thymeleaf dependency also includes the Spring MVC dependency.
It can write code in the HTML format and has high performance.
The devtools is a greate tool for Spring Boot. It can hot reload our code automatically.
The parent and build section are boilerplate part of Spring Boot Maven project.
You may need to config your own name for artifactId and groupId.

Controller Class

Implement a Spring MVC controller

This is just a simple Spring MVC example, and there is no configuration for Spring MVC.
The Spring Boot already has the default configuration for you.

HTML page by Thymeleaf

This HTML page you can build on src/main/resources/templates/greeting.html

Application Class

This class is Spring Boot start point.
@SpringBootApplication include three anotation, @Configuration, @EnableAutoConfiguration, @ComponentScan.
Note that we have to make sure the other Class under the hello package. Otherwise, you need to config the
@ComponentScan especially.

Execute Spring Boot

According to the article I mentioned before. You can use the command line to execute our code. But, I never do it that way.
I just use IDE to run the code. However, if you want to deploy to production, you will need to learn how to run
Spring Boot on the command line. But, for now, I just keep it is simple. We execute it by IDE. How to? Just go to
Application.class and right click. Then, run it like a normal jar class.
Now, we do a simple test. You can go to the browser(or you can use curl command) and open http://localhost:8080/greeting. Then, you will see the result.

Conclusion

I think this is a simple example for a newbie. Next step I plan is to introduce Spring MVC.
